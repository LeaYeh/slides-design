## Claude Code Framework

**Profile: personal**

Quick fixes, daily pain points, wild ideas. Optimize for speed and a working result.

- Speed first; YAGNI to the max; one-shot it. Don't over-engineer.
- Skip ADRs, tests, and review **unless explicitly asked**.
- Reach for `/prototype` for throwaway exploration of an idea; `/diagnose` only when a bug genuinely needs it.
- Keep changes surgical (baseline `karpathy-guidelines` already enforces this).
